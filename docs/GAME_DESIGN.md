# EastFront 2026 — Game Design Document\n\n## Overview\n\nEastFront 2026 is a turn-based, hex-based operational-level strategy game simulating the opening phases of Operation Barbarossa (1941). Players command either Axis or Soviet forces, managing unit movement, combat, and strategic objectives across a stylized Eastern European map.\n\n## Design Philosophy\n\n- **Accessibility**: Easy to learn rules, quick turn resolution\n- **Strategic Depth**: Simple mechanics allow emergent gameplay\n- **Historical Flavor**: Inspired by Barbarossa, but simplified for rapid play\n- **Expandability**: Core rules are minimal; new features can be added incrementally\n\n---\n\n## Core Game Mechanics\n\n### Hex Coordinate System\n\n- **Type**: Axial coordinates (q, r)\n- **Orientation**: Pointy-top hexes\n- **Map Size**: ~40x40 hex grid (1,600 hexes)\n- **Hex Resolution**: ~50 km per hex\n\n### Unit System\n\n#### Unit Types\n\n| Type | Strength | Movement | Role |\n|------|----------|----------|------|\n| **Infantry** | 3 | 3 | Defensive, holds ground |\n| **Armor** | 4 | 4 | Offensive, breakthrough |\n| **Headquarters** | 2 | 2 | Support (future: supply) |\n\n#### Unit Properties\n\n- **Position**: Hex coordinate (q, r)\n- **Strength**: 1–5 (attack/defense power)\n- **Movement Points**: Remaining movement this turn\n- **Max Movement**: Reset each turn\n- **Owner**: Axis or Soviet\n\n### Turn Structure\n\n**One complete round:**\n\n1. **Axis Turn**: Axis player selects and moves units\n2. **Soviet Turn**: Soviet player selects and moves units\n3. **Repeat**\n\n**Per-Turn Phases:**\n\n1. **Select** — Click a unit to select it\n2. **Move** — Click destination hex (within movement range)\n3. **Attack** — Adjacent enemies (future: attack resolution)\n4. **End Turn** — Confirm and pass to next player\n\n### Movement\n\n- Units move within their **Movement Points** each turn\n- Each hex costs 1 movement point\n- Valid moves are highlighted on the map\n- Units reset movement points at the start of their turn\n\n### Combat (Simplified Version 1)\n\n**Future expansion**: Odds-based attack resolution\n\n- Attacker strength vs. Defender strength\n- 2:1+ odds = guaranteed hit\n- 1:1 odds = 50% hit\n- Loser retreats 1 hex or is eliminated\n\n**Current v1**: No combat resolution yet (units can occupy same hex)\n\n### Terrain\n\n| Terrain | Effect | Color |\n|---------|--------|-------|\n| **Plain** | No penalty | Light Green |\n| **Forest** | -1 Defense (future) | Dark Green |\n| **Mountain** | -1 Movement (future) | Gray |\n| **Water** | Impassable | Blue |\n| **City** | Strategic objective | Gold |\n\n---\n\n## Barbarossa Scenario\n\n### Setting\n\n- **Year**: 1941, Summer\n- **Duration**: 10 turns (representing ~6 weeks)\n- **Map Focus**: Eastern Europe (Poland, Western USSR, Western Ukraine)\n\n### Initial Setup\n\n#### Axis Forces\n\n- **Deployment**: Western edge of map\n- **Composition**:\n  - 1 Armor (Headquarters)\n  - 4 Armor divisions\n  - 6 Infantry divisions\n  - 1 Armor (Reconnaissance)\n\n#### Soviet Forces\n\n- **Deployment**: Eastern edge of map\n- **Composition**:\n  - 1 Armor (Army HQ)\n  - 2 Armor divisions\n  - 8 Infantry divisions\n  - Strategic positions around Moscow, Leningrad, Kiev\n\n### Victory Conditions\n\n#### Axis Victory\n\n- Control Moscow by turn 10 (**Decisive Victory**)\n- Control 3+ major cities by turn 10 (**Major Victory**)\n- Eliminate 50% of Soviet forces (**Minor Victory**)\n\n#### Soviet Victory\n\n- Prevent Axis from controlling Moscow (**Defensive Victory**)\n- Eliminate Axis HQ (**Tactical Victory**)\n- Eliminate 70% of Axis forces (**Major Victory**)\n\n#### Draw\n\n- 10 turns elapse without victory conditions met\n\n---\n\n## User Interface (UI)\n\n### Main Screen Layout\n\n```\n┌─────────────────────────────────────────┐\n│  EastFront 2026  Turn: 1  Axis's Turn  │\n├─────────────────────────────────────────┤\n│                          │               │\n│      HEX MAP CANVAS      │   SIDEBAR    │\n│   (1200 x 800px)         │              │\n│                          │ Status       │\n│                          │ Unit Info    │\n│                          │ Controls     │\n│                          │              │\n└─────────────────────────────────────────┘\n```\n\n### Sidebar Panels\n\n1. **Game Status**\n   - Current player\n   - Turn number\n   - Last action description\n\n2. **Selected Unit Info**\n   - Unit type (Infantry/Armor/HQ)\n   - Strength (1–5)\n   - Current movement (X/Y remaining)\n   - Position (q, r)\n\n3. **Controls**\n   - End Turn button\n   - Reset Game button\n\n### Canvas Rendering\n\n- **Hexes**: Terrain color + border\n- **Units**: Colored circle (Axis=Red, Soviet=Blue) with type label (I/A/HQ)\n- **Valid Moves**: Yellow highlight\n- **Cities**: Gold hex with label\n\n---\n\n## Development Roadmap\n\n### Phase 1: Project Setup ✅\n- Vite + TypeScript scaffold\n- Basic Coordinate math, hex utilities\n├── map.ts            # Map generation, terrain data\n├── game.ts           # GameState, Unit, Player, TurnPhase\n└── ai.ts             # AI decision logic (stub)\n```\n\n### Key Classes\n\n#### `GameState`\n- Manages game state (turn, player, units)\n- Executes turn logic (select, move, endTurn)\n- Queries unit positions and valid moves\n\n#### `Unit`\n- Properties: id, type, owner, position, strength, movement\n- No methods; mutable state\n\n#### `AIPlayer` (Future)\n- Encapsulates AI decision-making\n- Takes game state and outputs moves\n\n---\n\n## Future Enhancements\n\n### Short-term (v0.2–v0.3)\n\n- [ ] Combat resolution system\n- [ ] Supply/logistics mechanics\n- [ ] Fog of war\n- [ ] Unit destruction/casualties\n- [ ] **HQ**: 2 hexes/turn (slower, support role)\n\n### Strength\n\n- **Infantry**: 3 (medium)\n- **Armor**: 4 (strong)\n- **HQ**: 2 (weak, support)\n\n### Deployment Imbalance\n\n- **Axis**: Fewer units, higher individual strength (blitzkrieg)\n- **Soviet**: More units, spread out (defensive depth)\n- **Intended**: Axis has early advantage; game tightens as Soviets consolidate\n\n---\n\n## Testing
